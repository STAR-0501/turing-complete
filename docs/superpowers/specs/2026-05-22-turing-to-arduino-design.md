# Turing Complete → Arduino 转换器 — 设计规范

**日期:** 2026-05-22
**状态:** 草案

## 1. 概述

将 Turing Complete 数字逻辑模拟器中搭建的组合逻辑电路，自动转化为 Arduino C++ 代码，通过 arduino-cli 编译并上传到 Arduino Uno 开发板。

## 2. 范围

### 2.1 支持的电路类型
- **组合逻辑电路**（Combinational Logic）— 无反馈环路、无状态存储
- 支持的元件：AND、OR、NOT、INPUT、OUTPUT、FUNCTION（自定义模块，递归展开内联）
- **不支持的电路：** 包含反馈环路/时序逻辑的电路（如 D 触发器、计数器、寄存器）

### 2.2 目标硬件
- **主要目标：** Arduino Uno (ATmega328P)
- 引脚映射：INPUT → D2-D7 (最多 6 个)，OUTPUT → D8-D13 (最多 6 个)
- 若 INPUT + OUTPUT > 12，报错提示引脚不足

### 2.3 输出产物
- Arduino `.ino` 文件（sketch）
- 通过 arduino-cli 编译生成的 `.hex` 二进制
- 上传到已连接的 Arduino 开发板

## 3. 架构

```
circuit_data.json
       │
       ▼
┌─────────────────┐
│  circuit_parser │  ← 解析电路 JSON，构建 DAG
│  .py            │
└────────┬────────┘
         │ CircuitDAG (inputs, outputs, gates sorted)
         ▼
┌─────────────────┐
│  code_generator │  ← 拓扑排序 + 信号流变量生成 C++
│  .py            │
└────────┬────────┘

         ▼
┌─────────────────┐

│                 │
│  ┌───────────┐  │
│  │arduino-cli│  │  ← 子进程调用 (Go CLI)
│  │ (已安装)  │  │
│  └───────────┘  │
└────────┬────────┘
         │
         ▼
   Arduino Uno
   (物理硬件)
```

## 4. 组件详细设计

### 4.1 `circuit_parser.py`

**输入:** `circuit_data.json` (Turing Complete 标准格式)
**输出:** `CircuitDAG` 数据结构

关键逻辑：
1. 读取 `elements[]` 和 `wires[]`
2. 建立 element_id → element 的字典映射
3. 对每条 wire，解析 start/end 的 elementId + portId，建立连接关系
4. 识别 INPUT 和 OUTPUT 类型的边界元件
5. 对逻辑门（AND/OR/NOT）进行拓扑排序（Kahn 算法）
6. 若检测到环（非 DAG），抛出错误

```python
@dataclass
class CircuitNode:
    id: str
    type: str          # AND/OR/NOT/INPUT/OUTPUT
    alias: str         # 用户指定的别名
    inputs: List[str]  # 输入源节点的 ID 列表
    expression: str    # 生成的 C++ 表达式

@dataclass
class CircuitDAG:
    inputs: List[CircuitNode]     # 按 alias 排序
    outputs: List[CircuitNode]    # 按 alias 排序
    gates: List[CircuitNode]      # 拓扑排序后的逻辑门
    max_intermediate: int         # 使用的中间变量数量
    pin_map: Dict[str, int]       # alias → Arduino 引脚号
```

### 4.2 `code_generator.py`

**输入:** `CircuitDAG` + `pin_map`
**输出:** `.ino` 文件（sketch）

**代码生成策略 — 信号流变量法（方案 A）：**

对每个逻辑门生成一个中间布尔变量，按拓扑顺序求值：

```
# 伪代码
for each gate in topology_sorted_gates:
    if gate.type == "AND":
        emit(f"bool t{n} = {inputs[0]} && {inputs[1]};")
    elif gate.type == "OR":
        emit(f"bool t{n} = {inputs[0]} || {inputs[1]};")
    elif gate.type == "NOT":
        emit(f"bool t{n} = !{inputs[0]};")
```

**FUNCTION（自定义模块）处理：**
1. 从 `modules_data.json` 查找模块定义
2. 递归展开：将模块内电路内联到主电路中
3. 模块的 INPUT 元件 → 连接到调用处的输入变量
4. 模块的 OUTPUT 元件 → 映射回函数输出的变量名

**生成的 .ino 文件结构：**
```cpp
// Auto-generated by Turing Complete → Arduino Converter

// Pin definitions
const int PIN_<ALIAS> = <pin>;

void setup() {
    // pinMode() for all INPUTs and OUTPUTs
}

void loop() {
    // Read all INPUTs: bool <alias> = digitalRead(PIN_<ALIAS>);
    // Evaluate gates in topological order
    // Write all OUTPUTs: digitalWrite(PIN_<ALIAS>, <value>);
    delay(10);
}
```

### 4.3 `uploader.py`

**主要接口：**
```python
def compile_sketch(sketch_dir: str, fqbn: str) -> bool
def detect_boards() -> List[BoardInfo]
def upload_sketch(sketch_dir: str, fqbn: str, port: str) -> bool
```

**实现方式 — CLI 子进程调用 arduino-cli：**
```python
import subprocess

def compile_sketch(sketch_dir, fqbn="arduino:avr:uno"):
    result = subprocess.run(
        ["arduino-cli", "compile", "--fqbn", fqbn, sketch_dir],
        capture_output=True, text=True
    )
    return result.returncode == 0, result.stdout, result.stderr

def detect_boards():
    result = subprocess.run(
        ["arduino-cli", "board", "list", "--format", "json"],
        capture_output=True, text=True
    )
    return json.loads(result.stdout)

def upload_sketch(sketch_dir, fqbn, port):
    result = subprocess.run(
        ["arduino-cli", "upload", "--fqbn", fqbn, "--port", port, sketch_dir],
        capture_output=True, text=True
    )
    return result.returncode == 0, result.stdout, result.stderr
```

**备选方案 — gRPC 调用：**
- 编译 arduino-cli 为 Go gRPC 服务
- 通过 gRPC 客户端调用 Compile/Upload API
- 需要 Go 编译环境，复杂度较高
- 推荐在 CLI 方式不满足需求时使用

## 5. 引脚映射策略

| 电路元件类型 | Arduino Uno 引脚 | 说明 |
|-------------|-----------------|------|
| INPUT 1 | D2 | 第一个输入 |
| INPUT 2 | D3 | 第二个输入 |
| ... | ... | 依次分配 |
| INPUT N | D(N+1) | 最大到 D7 |
| OUTPUT 1 | D8 | 第一个输出 |
| OUTPUT 2 | D9 | 第二个输出 |
| ... | ... | 依次分配 |
| OUTPUT M | D(M+7) | 最大到 D13 |

约束：
- 最大 INPUT 数：6（D2-D7）
- 最大 OUTPUT 数：6（D8-D13）
- 总引脚数约束：12 个可用数字引脚

## 6. 错误处理

| 错误场景 | 处理方式 |
|----------|---------|
| 电路包含反馈环路 | 检测到环后报错，提示「不支持的时序电路」 |
| INPUT/OUTPUT 超过 12 个 | 报错「引脚数不足，请减小电路规模」 |
| arduino-cli 未安装 | 提示用户安装，输出安装指引链接 |
| 无 Arduino 板连接 | `board list` 返回空列表，提示连接硬件 |
| 编译失败 | 返回 arduino-cli 的编译错误输出 |
| 上传失败 | 返回 arduino-cli 的上传错误输出 |

## 7. 集成方式

### 7.1 作为独立 CLI 工具
```bash
python turing-to-arduino.py --circuit circuit_data.json --output ./sketch
```

### 7.2 集成到 Turing Complete 后端
- 在 `app.py` 中新增 `POST /api/export/arduino` 路由
- 调用转换器生成 .ino 文件
- 返回下载链接或直接触发上传

### 7.3 AI Agent 集成
- 新增 AI 命令 `EXPORT_ARDUINO`
- AI Agent 可在电路构建完成后自动触发导出
- 通过 `AI_INSTRUCTIONS.md` 添加新指令

## 8. 依赖

- Python 3.8+
- arduino-cli（用户自行安装）
  - 安装指引：https://arduino.github.io/arduino-cli/latest/installation/
  - 初始化：`arduino-cli core install arduino:avr`
- 无额外 Python 依赖（仅标准库）

## 9. 局限性与未来工作

### 当前局限
- 仅支持组合逻辑，不支持时序/状态电路
- 仅支持 Arduino Uno（FQBN: `arduino:avr:uno`）
- 不支持模拟引脚（A0-A5）
- 最大 12 个 I/O 引脚

### 可能的扩展方向
- [ ] 支持含状态电路（引入定时器采样、移位寄存器模式）
- [ ] 支持更多 Arduino 板型（Mega、Nano、Leonardo）
- [ ] 支持模拟输入/输出（PWM/ADC）
- [ ] 交互式引脚映射编辑器
- [ ] 生成接线图（辅助物理连线）
