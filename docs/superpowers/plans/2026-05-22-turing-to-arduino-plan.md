### 步骤 2：实现 `circuit_parser.py`

**文件：** `turing_to_arduino/circuit_parser.py`

**职责：** 读取 `circuit_data.json`，解析为 `CircuitDAG` 结构

**关键实现细节：**

1. 数据结构：
```python
@dataclass
class CircuitNode:
    id: str
    type: str          # AND/OR/NOT/INPUT/OUTPUT
    alias: str
    inputs: list[str]  # 连接到本节点 input 端口的源节点 ID 列表
    output_to: list[str]  # 本节点输出连接到哪些节点

@dataclass
class CircuitDAG:
    inputs: list[CircuitNode]     # 按 alias 排序
    outputs: list[CircuitNode]    # 按 alias 排序
    gates: list[CircuitNode]      # 拓扑排序后的逻辑门
    pin_map: dict[str, int]       # alias → Arduino 引脚号
```

2. `parse_circuit(data: dict) -> CircuitDAG`:
   - 建立 element_id → element 映射
   - 解析 wires：对每条 wire，确定 source element 和 target element
   - 对每个 element，根据 wires 确定其输入源（连接到其 input 端口的其他 element）
   - 将 INPUT 元素加入 `inputs` 列表，OUTPUT 加入 `outputs` 列表
   - 对 AND/OR/NOT/FUNCTION 执行 Kahn 拓扑排序
   - 若排序后还有剩余节点 → 检测到环 → 抛出 `ValueError("Circuit contains feedback loop")`
   - 自动生成 `pin_map`：inputs 按顺序分配 D2-D7，outputs 分配 D8-D13

3. **FUNCTION 模块处理：**
   - 模块定义存储在 `modules_data.json`（由调用方传入）
   - 遇到 FUNCTION 类型 element，从模块定义中加载其内部 elements 和 wires
   - 递归展开：将模块内电路内联，模块 INPUT 对应调用处的输入变量，模块 OUTPUT 对应新的中间变量
   - 展开后的节点同样参与拓扑排序

4. **验证：** 在当前 `circuit_data.json` 上测试解析，验证 HalfAdder（8 inputs, 5 outputs, gates 约 20+）能正确拓扑排序

### 步骤 3：实现 `code_generator.py`

**文件：** `turing_to_arduino/code_generator.py`

**职责：** 将 `CircuitDAG` 生成为 Arduino .ino sketch

**核心逻辑（方案 A — 信号流变量法）：**

```python
def generate_arduino_sketch(dag: CircuitDAG) -> str:
    lines = []
    lines.append("// Auto-generated by Turing Complete → Arduino Converter")
    lines.append("")
    
    # 引脚宏定义
    for alias, pin in dag.pin_map.items():
        lines.append(f"const int PIN_{alias.upper()} = {pin};")
    lines.append("")
    
    # setup()
    lines.append("void setup() {")
    for node in dag.inputs:
        alias = node.alias or f"IN{node.id[:4]}"
        pin = dag.pin_map.get(alias, 2)
        lines.append(f"  pinMode(PIN_{alias.upper()}, INPUT);")
    for node in dag.outputs:
        alias = node.alias or f"OUT{node.id[:4]}"
        pin = dag.pin_map.get(alias, 8)
        lines.append(f"  pinMode(PIN_{alias.upper()}, OUTPUT);")
    lines.append("}")
    lines.append("")
    
    # loop()
    lines.append("void loop() {")
    
    # 读输入
    for node in dag.inputs:
        alias = node.alias or f"IN{node.id[:4]}"
        pin = dag.pin_map.get(alias)
        lines.append(f"  bool {_safe_name(alias)} = digitalRead(PIN_{alias.upper()});")
    
    # 按拓扑顺序求值
    var_counter = 1
    var_map = {}  # element_id → variable name
    for node in dag.gates:
        # 获取输入变量的名称
        input_vars = []
        for src_id in node.inputs:
            if src_id in var_map:
                input_vars.append(var_map[src_id])
            else:
                # 检查是否是 INPUT 节点
                src_node = find_node(dag, src_id)
                if src_node:
                    input_vars.append(_safe_name(src_node.alias or f"IN{src_node.id[:4]}"))
        
        # 生成表达式
        if node.type == "AND":
            expr = f" && ".join(input_vars) if input_vars else "false"
        elif node.type == "OR":
            expr = f" || ".join(input_vars) if input_vars else "false"
        elif node.type == "NOT":
            expr = f"!{input_vars[0]}" if input_vars else "false"
        elif node.type == "FUNCTION":
            expr = f" && ".join(input_vars) if input_vars else "false"
        else:
            continue
        
        var_name = f"t{var_counter}"
        var_counter += 1
        var_map[node.id] = var_name
        lines.append(f"  bool {var_name} = {expr};")
    
    # 写输出
    for node in dag.outputs:
        alias = node.alias or f"OUT{node.id[:4]}"
        # 找到连接到 OUTPUT 的源变量
        output_source = ...
        lines.append(f"  digitalWrite(PIN_{alias.upper()}, {output_source});")
    
    lines.append("  delay(10);")
    lines.append("}")
    
    return "\n".join(lines)
```

**细节注意：**
- 变量名 `_safe_name()` 将 alias 中的特殊字符替换为下划线
- 每个 FUNCTION 模块展开时，内部节点分配独立的变量计数器
- 输出节点的值通过查找谁连接到它的输入端口来确定

### 步骤 4：实现 `uploader.py`

**文件：** `turing_to_arduino/uploader.py`

**职责：** 封装 arduino-cli 的子进程调用

```python
import subprocess, json, shutil
from pathlib import Path

ARDUINO_CLI = "arduino-cli"  # 假设在 PATH 中

def check_arduino_cli() -> bool:
    """检查 arduino-cli 是否已安装"""
    return shutil.which(ARDUINO_CLI) is not None

def compile_sketch(sketch_dir: str, fqbn: str = "arduino:avr:uno") -> tuple[bool, str, str]:
    """编译 sketch，返回 (成功, stdout, stderr)"""
    result = subprocess.run(
        [ARDUINO_CLI, "compile", "--fqbn", fqbn, sketch_dir],
        capture_output=True, text=True, timeout=120
    )
    return result.returncode == 0, result.stdout, result.stderr

def detect_boards() -> list[dict]:
    """检测已连接的 Arduino 板，返回列表"""
    result = subprocess.run(
        [ARDUINO_CLI, "board", "list", "--format", "json"],
        capture_output=True, text=True, timeout=30
    )
    if result.returncode != 0:
        return []
    return json.loads(result.stdout)

def upload_sketch(sketch_dir: str, port: str, fqbn: str = "arduino:avr:uno") -> tuple[bool, str, str]:
    """上传到指定端口，返回 (成功, stdout, stderr)"""
    result = subprocess.run(
        [ARDUINO_CLI, "upload", "--fqbn", fqbn, "--port", port, sketch_dir],
        capture_output=True, text=True, timeout=60
    )
    return result.returncode == 0, result.stdout, result.stderr
```

### 步骤 5：实现 `cli.py`

**文件：** `turing_to_arduino/cli.py`

命令行入口，支持两种模式：

```bash
# 模式 1：仅生成代码
python -m turing_to_arduino --circuit circuit_data.json --output ./my_sketch

# 模式 2：生成 + 编译 + 上传
python -m turing_to_arduino --circuit circuit_data.json --upload --port COM3
```

使用 `argparse` 标准库：
```
--circuit PATH     必需：电路 JSON 文件路径 (默认: circuit_data.json)
--modules PATH     可选：模块数据 JSON (默认: modules_data.json)
--output PATH      可选：输出 sketch 目录 (默认: ./output/sketch)
--upload           可选：编译后上传
--port PORT        可选：上传目标端口 (如 COM3, /dev/ttyACM0)
--fqbn FQBN        可选：目标板 FQBN (默认: arduino:avr:uno)
--list-ports       可选：列出已连接板子
```

**验证：** `python -m turing_to_arduino --list-ports` 运行成功
